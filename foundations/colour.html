          Colour

          This page defines the colour roles used across Gen 2 Media Sites and how those roles are expressed in different contexts.

          Overview

          This page defines the colour roles used across Gen 2 Media Sites and how those roles are expressed in different contexts.

          Gen 2 deliberately separates:

          • Roles – what a colour is for
          • Contexts – where that colour appears

          This separation allows the system to remain consistent while adapting to different visual environments, such as banners or media headers.

          This page defines colour roles and contexts only.
          How colours are mapped per Media Site is documented under Foundations → Themes.

          Colour roles

          Gen 2 uses three core colour roles:

          • Base – structure and text
          • Action – interaction and affordance
          • Semantic – meaning and system feedback

          Roles define intent and behaviour.
          Roles do not change between contexts.

          Colour contexts

          Colours are rendered within different contexts.
          Contexts define the visual environment in which colours are applied and may influence how roles are visually expressed.

          Gen 2 supports two primary contexts:

          • Standard context – typical page surfaces
          • Emphasis context – highlighted surfaces such as banners or media headers

          Contexts may adjust appearance to maintain contrast and clarity, but do not introduce new roles or meanings.

          Base role

          Purpose

          The Base role defines the structural foundation of the interface.
          Base colours prioritise legibility, hierarchy and stability and make up the majority of the UI.

          Used for

          • Page backgrounds
          • Content surfaces (cards, panels)
          • Headers and footers
          • Borders and dividers
          • Primary, secondary and muted text

          Base in different contexts

          Base (standard context)

          • Used for typical page surfaces
          • Provides neutral contrast for content and actions
          • Remains visually unobtrusive

          Base (emphasis context)

          • Used for highlighted surfaces such as banners and media headers
          • Visually distinct from standard surfaces
          • Does not convey interaction or meaning

          Base remains non-interactive in all contexts.

          Base naming

          Base colours are named by luminance, not hue.

          • Lower numbers represent lighter colours
          • Higher numbers represent darker colours
          • Only Base steps intended for direct use are documented

          Base colour roles

          Base colours are documented only where they represent a deliberate design decision.

          Base step Intended use
          Base 00 Page background
          Base 01 Primary surface (content areas, cards)
          Base 02 Elevated surface (modals, overlays, raised sections)
          Base 03 Border and divider colour
          Base 60 Muted or secondary text
          Base 80 Primary body text
          Base 90 High-emphasis text (for example headings on light surfaces)

          Additional internal Base steps may exist for refinement but are not intended for direct use unless explicitly documented.

          Action role

          Purpose

          The Action role defines interaction and affordance.

          Actions answer the question:
          What can I interact with?

          Used for

          • Buttons
          • Links
          • Icon actions (for example favourite, add to list)
          • Focus indicators
          • Active and selected states

          Action in different contexts

          Action (standard context)

          • Used on typical page surfaces
          • Clear visual affordance
          • Supports primary and secondary interaction patterns

          Action (emphasis context)

          • Used within emphasis regions such as banners or media headers
          • Retains the same role and meaning as standard context actions
          • May use a contrast-adjusted visual treatment to remain clear against the emphasis surface

          Adjustments may include changes to fill, outline, elevation, shape or text colour.

          Context may change appearance, not semantics.

          Semantic role

          Purpose

          The Semantic role conveys meaning and system feedback.

          Used for

          • Success
          • Warning
          • Error
          • Info

          Semantic across contexts

          Semantic colours maintain consistent meaning across both standard and emphasis contexts.

          They must:

          • Remain recognisable
          • Maintain sufficient contrast
          • Never rely on colour alone to convey meaning

          Semantic colours are not themed.

          Accessibility and contrast

          All colour usage must support accessibility and readability.

          General guidance:

          • Text must maintain sufficient contrast against its surface
          • Actions must remain clearly distinguishable in all contexts
          • Emphasis surfaces must not reduce the clarity of content or interaction
          • Where possible, rely on Base text colours for long-form content

          Relationship to themes

          This page defines roles and contexts.

          How Base and Action roles are visually expressed per Media Site is documented under Foundations → Themes.

          Summary

          • Roles define intent
          • Contexts define environment
          • Appearance may adapt, meaning does not

          If a colour cannot be described by both a role and a context, it does not belong in the system.
